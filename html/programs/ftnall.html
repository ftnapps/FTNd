ftnall - Allfiles listing generator

Synopsis.

ftnall [commands] <options>

 

Description.

ftnall is the allfiles and newfiles listing generator that can be made available on your bbs for your users to get a complete listing of wat is available for download. When used with the -zip option it can also produce complessed versions of these two listings. The resulting files are created in the current directory. After the creation of these files you can hatch them into your bbs with the programs ftnfido tic when you properly setup a .tic file area for this purpose and create records for the hatch manager.

Just before the reports are created, and if you defined WWW and FTP logfiles with ftnsetup in menu 1.13, the logfiles are analysed and and files downloaded from the bbs are counted as valid downloads. To make sure downloads are only counted once, a status file is created in ~/var, this marks the date the last time the logfiles are checked. For the best results make sure you run ftnall before any WWW and FTP logfiles are rotated, else you might miss some downloads.

 

Environment.

In order to run ftnall you need to set the global variable $FTND_ROOT. This variable must point to the root of the bbs directory structure. The main configuration file config.data must be present in the ~/etc directory.

 

Commands.

ftnall list - Create allfiles.txt and newfiles.txt files.

 

Options.

ftnall [command] -quiet - Quiet mode, supress screen output.
ftnall list -zip - Create zipped listings as well.

 

Setup.

In ftnsetup menu 1.13 you need to set the days to include in newfiles listings and the maximum security level. Also define valid paths to logfiles that are created by your web and ftp server.
