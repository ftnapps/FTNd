ftndiff - Nodelist difference file processor.

Synopsis.

ftndiff [nodelist] [nodediff] <options>

 

Description.

ftndiff applies a (compressed) nodediff file against the nodelist of the week before to create a new nodelist. The result is a new plain nodelist and a nodelist compressed with zip.

 

Environmet.

In order to run ftndiff you must set the global variable $FTND_ROOT. This variable must point to the root directory of the bbs structure. The main configuration file config.data must be present in the ~/etc directory.

 

Commands.

ftndiff [nodelist] [nodediff] The nodelist must be the full path and filename without the dot and daynumber extension. The nodediff is the full path and filename to the (compressed) nodediff file fitting on the latest nodelist. It is adviced to make a seperate working directory where you keep the nodelists. Don't do this in your normal nodelist directory. When the operation is successfull, the new nodelist is in the working directory and the old list is removed. A compressed version of the nodelist is also placed in the working directory. From here you can hatch the new compressed nodelist with the ftnfido program.

 

Options.

-quiet - supress screen output, this switch is needed when ftndiff runs on the background.
